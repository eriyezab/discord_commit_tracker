# Discord Commit Tracker

A Discord bot that automatically posts to a designated channel whenever someone pushes a commit to a tracked GitHub repository. Designed for passive sharing — a small friend group can see what others are working on without anyone needing to remember to post.

Originally built to track progress on [100 Days of SwiftUI](https://www.hackingwithswift.com/100/), but works with any GitHub repo.

> **Note:** This is a self-hosted bot. You must host the code yourself and create your own bot in the Discord Developer Portal to use it. A publicly hosted version that anyone can add to their server without any setup is planned for a future release.

## How It Works

```
GitHub Webhooks → Webhook Receiver (FastAPI) → Discord Bot → Single Channel
```

When a tracked repo receives a push, GitHub sends a POST request to the bot's webhook endpoint. The bot verifies the signature, formats the commit data, and posts an embed to the configured Discord channel.

## Tech Stack

- **Python 3.11+**
- **discord.py** — Discord bot client
- **FastAPI + uvicorn** — Webhook receiver
- **SQLite + aiosqlite** — Persistence
- **Railway or Fly.io** — Hosting (must be publicly reachable)
- **ngrok** — Local development tunneling

## Setup

### Prerequisites

1. Create a Discord bot at the [Discord Developer Portal](https://discord.com/developers/applications) and invite it to your server.
2. Deploy to a publicly reachable host (Railway, Fly.io, etc.) or use ngrok for local development.

### Environment Variables

| Variable | Description |
|---|---|
| `DISCORD_TOKEN` | Bot token from the Discord Developer Portal |
| `DISCORD_CHANNEL_ID` | The channel the bot posts commit embeds to |
| `DISCORD_GUILD_ID` | Your Discord server ID |
| `DATABASE_PATH` | Path to the SQLite file (default: `./data/bot.db`) |
| `WEBHOOK_BASE_URL` | Public base URL of the bot's server (e.g. `https://yourbot.fly.dev`) |
| `PORT` | Port for the HTTP server (default: `3000`) |

### Adding a Repo

Use the `/track` slash command in Discord:

```
/track owner/repo
```

The bot will respond with a webhook URL and secret, plus step-by-step instructions for adding the webhook in your GitHub repo settings.

## Slash Commands

| Command | Description |
|---|---|
| `/track <owner/repo>` | Start tracking a repo; provides webhook setup instructions |
| `/untrack <owner/repo>` | Stop tracking a repo |
| `/list` | Show all currently tracked repos |

## Webhook Endpoint

`POST /github/webhook` — Receives push events from GitHub. Verifies the `X-Hub-Signature-256` header, filters to default branch pushes only, and posts one embed per commit.

`GET /health` — Health check endpoint.

## Discord Embed Format

Each commit is posted as an embed containing:

- Commit author name
- Commit message (first line, truncated to 80 chars)
- Repository name linked to the repo URL
- Link to the commit diff
- Commit timestamp
- Private repo notice (if applicable)

If a push contains more than 5 commits, the 5 most recent are posted with a link to the full push comparison.

## Security

- Each tracked repo gets a unique webhook secret generated by the bot.
- All incoming webhook requests are verified via HMAC-SHA256 before processing.
- Slash command responses containing secrets are ephemeral (only visible to the user who ran the command).

## Development

Use ngrok to expose your local server for testing:

```bash
ngrok http 3000
```

Set `WEBHOOK_BASE_URL` to your ngrok URL, then add a webhook to a test repo in GitHub and push a commit to verify the flow.

## Future Features

These are out of scope for v1 but planned for future releases:

- **GitHub-to-Discord user mapping** — link GitHub usernames to Discord users so commit embeds mention the right person via `/link` and `/unlink` commands.
- **LeetCode submission tracking** — poll LeetCode for accepted submissions and post them to the same channel as commits.
- **Streak tracking** — track consecutive days of commits and post milestone notifications (e.g. 7-day, 30-day streaks). Especially useful for challenge-based projects like 100 Days of SwiftUI.
- **Weekly digest** — a scheduled weekly summary of activity across all tracked repos, including commit counts and a leaderboard.
- **PR and issue notifications** — extend beyond push events to notify when a pull request is opened/merged or an issue is opened/closed.
- **Per-repo mute/unmute** — temporarily silence a repo without removing the webhook.
- **Public hosted version** — a centrally managed cloud instance that anyone can add to their Discord server without needing to host anything or create their own bot.
